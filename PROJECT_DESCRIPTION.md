# Описание проекта "Добрые Лапки" - Система управления приютом для животных

## 1. ВВЕДЕНИЕ

Проект "Добрые Лапки" представляет собой веб-приложение для управления приютом для животных. Система обеспечивает комплексное управление всеми аспектами работы приюта: от учета животных и обработки заявок на усыновление до финансового планирования и аналитики.

## 2. СТЕК ТЕХНОЛОГИЙ

### 2.1. Backend (Серверная часть)

**Язык программирования:** Python 3.6+

**Веб-фреймворк:** Flask 3.1.0
- Легковесный и гибкий фреймворк для создания веб-приложений
- Поддержка маршрутизации (routing)
- Обработка HTTP-запросов (GET, POST)
- Интеграция с шаблонизатором Jinja2

**Библиотеки и модули:**

1. **Flask-Login 0.6.3**
   - Управление сессиями пользователей
   - Аутентификация и авторизация
   - Декораторы для защиты маршрутов (@login_required)

2. **Werkzeug 3.1.3**
   - Хеширование паролей (generate_password_hash, check_password_hash)
   - Работа с файлами и загрузками
   - Утилиты для веб-разработки

3. **SQLite3** (встроенный в Python)
   - Реляционная база данных
   - Хранение всех данных приложения
   - Поддержка транзакций и внешних ключей

4. **Стандартные библиотеки Python:**
   - `datetime` - работа с датами и временем
   - `os` - работа с файловой системой
   - `json` - обработка JSON-данных

### 2.2. Frontend (Клиентская часть)

**HTML5 + CSS3**
- Семантическая разметка
- Современные CSS-технологии:
  - CSS Grid и Flexbox для верстки
  - CSS Variables (custom properties)
  - Градиенты и анимации
  - Адаптивный дизайн (responsive design)

**JavaScript (Vanilla JS)**
- Обработка событий DOM
- Работа с формами и модальными окнами
- AJAX-запросы к API
- Динамическое обновление интерфейса

**Библиотеки:**

1. **Chart.js 4.4.0**
   - Интерактивные графики и диаграммы
   - Поддержка различных типов визуализации:
     - Линейные графики (line charts)
     - Столбчатые диаграммы (bar charts)
     - Круговые диаграммы (doughnut charts)
   - Адаптивность и анимации

2. **Font Awesome 6.4.0**
   - Иконки для улучшения UX
   - Векторная графика

3. **Google Fonts (Inter)**
   - Современная типографика
   - Улучшенная читаемость

### 2.3. База данных

**SQLite 3**
- Легковесная встраиваемая СУБД
- Файловая база данных (shelter.db)
- Поддержка SQL-запросов
- ACID-совместимость

**Структура базы данных:**

1. **users** - Пользователи системы
   - id (PRIMARY KEY)
   - username (UNIQUE)
   - password_hash
   - role (admin, volunteer, guest)

2. **animals** - Животные в приюте
   - id (PRIMARY KEY)
   - name, description, status
   - age, gender, vaccinated, sterilized
   - photo_url

3. **adoption_requests** - Заявки на усыновление
   - id (PRIMARY KEY)
   - animal_id (FOREIGN KEY → animals)
   - user_name, phone, email
   - submitted_at, status (pending, approved, rejected)

4. **donations** - Пожертвования
   - id (PRIMARY KEY)
   - user_name, amount
   - donation_type (shelter, animal)
   - animal_id (FOREIGN KEY → animals, nullable)
   - donated_at

5. **inventory** - Инвентарь приюта
   - id (PRIMARY KEY)
   - item_name, category (food, medicine, supplies, other)
   - quantity, unit, unit_price
   - added_at

6. **news** - Новости приюта
   - id (PRIMARY KEY)
   - title, content, date

7. **stray_animals** - Бездомные животные на карте
   - id (PRIMARY KEY)
   - description, lat, lon
   - photo_url

8. **logs** - Логи действий пользователей
   - id (PRIMARY KEY)
   - action, description
   - user_id (FOREIGN KEY → users)
   - created_at

   - Детализация по категориям

   - Разбивка: на приют / на животное
   - Статистика по каждому типу

### 4.4. API Endpoints

**RESTful API для аналитики:**

1. **GET /api/analytics/donations-timeline?days=30**
   - Возвращает данные о пожертвованиях за период
   - Формат: JSON
   - Поля: date, total, count

2. **GET /api/analytics/adoption-trends?days=30**
   - Возвращает тренды усыновлений
   - Формат: JSON
   - Поля: date, total, approved, pending, rejected

3. **GET /api/analytics/inventory**
   - Возвращает анализ инвентаря по категориям
   - Формат: JSON
   - Поля: category, item_count, total_quantity, total_value, avg_price

**Безопасность API:**
- Требуется аутентификация (@login_required)
- Доступ только для администраторов
- Возврат 403 при отсутствии прав

## 5. АНАЛИТИЧЕСКИЕ ВОЗМОЖНОСТИ

### 5.1. Прогнозирование затрат

**Алгоритм прогнозирования:**

1. Сбор исторических данных за последние 90 дней
2. Расчет среднего дневного расхода
3. Вычисление тренда методом линейной регрессии
4. Прогнозирование на 3 месяца вперед

**Формула:**
```
predicted_cost = avg_daily * 30 + (trend * 30 * month_number)
```

**Особенности:**
- Работает без внешних библиотек (numpy не требуется)
- Учитывает тренд изменения расходов
- Защита от отрицательных значений

### 5.2. Финансовый анализ

**Метрики:**
- **Общие доходы** - сумма всех пожертвований
- **Общие расходы** - стоимость всего инвентаря
- **Баланс** - разница между доходами и расходами
- **Месячные показатели** - статистика за последние 30 дней

### 5.3. Статистический анализ

**Анализ усыновлений:**
- Процент одобренных заявок
- Среднее время обработки
- Тренды по периодам

**Анализ пожертвований:**
- Средняя сумма пожертвования
- Популярность типов пожертвований
- Динамика по времени

**Анализ инвентаря:**
- Распределение по категориям
- Средняя стоимость единицы
- Общая стоимость запасов
